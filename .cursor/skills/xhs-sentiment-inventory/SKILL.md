# 小红书关键词舆情清单处理

把「搜索结果截图 → 帖子全量清单 → 监测记录表统计」做成可复用流程。规则优先机械对齐与冲突检测；情感/日期/正文仍以人工复核为准。

## 何时使用

- 新项目：从空表模板建监测记录表，按关键词截图批量判帖
- 中途：跨屏合并、负向列表 `*N`、从清单重算主表
- 质检：标题/作者归一化探测、跨样本判定冲突清单

## 交付物结构（默认）

| Sheet | 作用 |
|-------|------|
| 监测记录表 | 一人一行；各关键词：总笔记数、负向数、提及数、负向率、笔记名称 |
| 填表说明 | 口径摘要 |
| 帖子全量清单 | 每条可见笔记一行（不含已合并的续接半卡） |
| 负向帖子重复度汇总 | 负向帖跨用户聚合 |
| 高频内容与历史问题 | 主题归类 |

笔记名称格式：`作者《标题》`；同用户完整重复：`作者《标题》*2`；无负向：`/`。  
鸿蒙负向标签若需日期：`作者《标题》日期` 或 `作者《标题》日期待确认`。

## 标准工作流

复制并跟踪：

```
- [ ] 1. 对齐模板关键词区块与样本图命名
- [ ] 2. 读截图建帖子全量清单（含跨屏合并）
- [ ] 3. 情感判定（正文优先；搜不到从严）
- [ ] 4. 归一化探测 + 跨样本判定冲突表
- [ ] 5. 人工复核清单（作者/标题/判定/日期）
- [ ] 6. 从清单重算主表与汇总 sheet
- [ ] 7. 备份命名；Excel↔Numbers 注意兼容
```

### 1. 样本与关键词

- 一人一文件夹 = ID；文件名常含 `ID_组号_关键词_屏号`
- 只处理用户指定的关键词组（例：nova 16 = `*_01_*`，鸿蒙系统 = `*_06_*`，各 3 屏）
- 双列瀑布流：左右列分别纵向看跨屏边界（屏1→2、屏2→3）

### 2. 情感判定

详见 [reference.md](reference.md)。摘要：

1. 能确认正文 → **以正文态度为准**（标题党不记负向）
2. 搜不到 → 仅标题+封面同为明确劝退/投诉才「确认负向」；否则「待复核」不记负向
3. 系列内对比仍留本品 → 中性，不记
4. 最终推竞品 → 负向
5. 竞品/配件/代言广告、搜索热词小卡 → 一般不记负向；热词标「搜索热词(非笔记)」

### 3. 跨屏合并与计数

**先合并半卡，再计数：**

- 上一屏底部半卡 + 下一屏顶部续接 = **同一帖**，只保留主记录
- 续接片段不计入负向、不进全量清单；主记录备注「已合并跨屏续接」
- 仅**独立完整重复卡**才标 `*2` / `*3`；跨屏半卡不算重复

**计数口径：**

- 负向数按屏（合并后续接后）统计，**不做用户内去重列**；完整重复用 `*N` 表达
- 不同用户搜到同一帖分别计入
- 负向率 = 负向笔记数 ÷ 总笔记数
- 总笔记数不含「搜索热词(非笔记)」
- 提及数一般不含热词、纯竞品广告、明显无关帖

### 4. 标题 / 作者归一化

目标：避免同帖被拆成多帖、同帖被判成多种态度。

- 聚合前：NFKC、去多余空格、统一全半角/常见标点、去掉 `…`/`...` 噪声后再比
- 裁切标题/作者：用同帖其他完整样本回填；补不全保留「裁切/未知」，不强猜
- 跨样本一致性：按「归一化作者 + 归一化标题」聚合同帖 → 检查判定/是否记入负向是否一致 → 冲突进待复核表 → 最终同帖共用一套作者、标题、判定、日期

跑探测：

```bash
python ~/.cursor/skills/xhs-sentiment-inventory/scripts/probe_inventory.py \
  --xlsx "/path/to/workbook.xlsx" \
  --sheet "帖子全量清单" \
  --out "/path/to/归一化探测.xlsx"
```

### 5. 从清单重算

人工复核清单后，**以帖子全量清单为唯一真相**重算：

- 监测记录表各关键词统计与笔记名称
- 负向帖子重复度汇总
- 高频内容与历史问题

项目内可有专用脚本（如 `recalculate_numbers.py`）；无脚本则按 reference 中的分组逻辑用 openpyxl / numbers_parser 重写。不要用旧 JSON 覆盖已复核清单。

### 6. 备份与兼容

改表前备份，文件名带阶段，例如：

- `*_调整前备份.xlsx`
- `*_跨屏复核前备份.xlsx`
- `*_删除去重列前备份.xlsx`

Numbers 不支持部分 Excel 条件格式：改用静态底色。导出 Excel 后删掉「导出摘要」类多余 sheet，清理浮点尾巴。

## 人机分工

| 适合规则/脚本 | 适合人工 |
|---------------|----------|
| 空格/标点归一、裁切作者回填、冲突检出、`*N`、从清单重算 | 情感最终定案、日期、正文歧义、是否同一帖的边界案例 |

不要声称规则可替代人工情感复核。
