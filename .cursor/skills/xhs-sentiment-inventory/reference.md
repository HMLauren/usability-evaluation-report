# 小红书舆情清单 — 详细规则

## 清单字段（帖子全量清单）

建议列顺序：

| 列 | 说明 |
|----|------|
| ID | 样本用户 ID |
| 关键词 | 如 `nova 16` / `鸿蒙系统` |
| 屏号 | `1`/`2`/`3` |
| 位置 | 如 `左上`/`右中`；便于跨屏核对 |
| 作者 | 可见作者；裁切写 `（裁切）` 等 |
| 标题 | 可见标题 |
| 内容类型 | `笔记` / `搜索热词(非笔记)` / `竞品广告` / `配件广告` / `代言营销` 等 |
| 判定状态 | `确认负向` / `待复核` / `中性` / `正向` / `无关` 等 |
| 是否记入负向 | `是` / `否` |
| 备注 | 跨屏续接、搜正文结论、人工说明 |

已合并的续接半卡：**不写入**清单。

## 情感判定细则

| 情形 | 判定 | 记入负向 |
|------|------|----------|
| 标题惨、正文吹 / 欲扬先抑 | 按正文 | 否（偏正/中性） |
| 系列内对比，结论仍留本品 | 中性 | 否 |
| 对比后推荐其他品牌/机型 | 确认负向 | 是 |
| 明确劝退、翻车、避雷本品/系统 | 确认负向 | 是 |
| 标题党 / 正负混杂 / 纯提问 | 待复核（搜不到正文时） | 否 |
| 竞品广告、配件广告、代言 | 对应类型 | 一般否 |
| 大家都在搜热词小卡 | 搜索热词(非笔记) | 否 |

搜正文：尽量搜；被墙或搜不到时严格执行「从严不记」，标待复核交给人。

## 跨屏续接判定启发

对每个 ID、每个关键词、每个边界（1→2、2→3），左右列分别看：

- 屏 N 最底部卡片残缺（作者/标题截断）且屏 N+1 顶部同列卡片标题/作者可对齐 → 续接
- 两屏均为完整卡、内容相同 → 独立重复，可 `*N`
- 仅作者缺失、标题完整相同 → 优先回填作者，再判断是续接还是重复（结合位置：底+顶更像续接）

## 归一化函数（与探测脚本一致）

```python
import re, unicodedata

def norm_title(value: object) -> str:
    text = unicodedata.normalize("NFKC", str(value or "")).lower()
    text = text.replace("…", "").replace("...", "").replace("·", "")
    return re.sub(r"[^0-9a-z\u4e00-\u9fff]+", "", text)

def norm_author(value: object) -> str:
    text = norm_title(value)
    if not text or any(m in text for m in ("裁切", "未知", "未显示", "未识别", "广告")):
        return ""
    return text
```

同帖聚类：先 `norm_title`；再在同标题下合并「已知作者 + 空作者」；多个不同已知作者则分拆（除非人工确认同人改名）。

标题变体组：同一 `norm_title` 下原始标题字符串种类 > 1，或已知作者字符串种类 > 1。

判定冲突：同组内 `判定状态` 集合大小 > 1；负向冲突：同组内 `是否记入负向` 同时有「是」和「否」。

## 笔记名称与 `*N`

同一 ID、同一关键词下，负向主记录按同帖合并：

- 出现 1 次：`作者《标题》`
- 完整独立出现 N 次：`作者《标题》*N`
- 作者未知：`《标题》` 或 `《标题》*N`
- 多条之间换行；单元格无负向填 `/`

不要把跨屏续接计进 N。

## 负向重复度汇总

对「是否记入负向=是」的行聚类后输出：

- 作者、帖子标题（取众数/最长完整标题）
- 覆盖用户数、屏幕出现次数、同用户跨屏重复次数、用户覆盖率、命中 ID 列表

高频可用静态底色，避免 Numbers 丢掉 Excel 条件格式。

## 主题归类（示例，可按品类改）

**机型类（如 nova 16）：** 定价高/高价低配；续航充电；配件适配；销量/市场反馈；新机体验/质量问题。

**系统类（如鸿蒙）：** 自动更新/数据问题；游戏/办公/校园场景受限；应用不适配/功能缺失；Bug/稳定性；升级后悔/回退；生态成熟度/长期观望；系统难用/交互体验。

用标题关键词命中归类；未命中进默认桶。

## 从清单重算主表

对每个 ID × 关键词：

1. 总笔记数 = 清单中该组且非搜索热词的行数
2. 负向笔记数 = `是否记入负向=是` 的行数（续接已不在清单）
3. 提及笔记数 = 按项目约定过滤广告/无关后的行数
4. 负向率 = 负向 / 总笔记（分母为 0 则空或 `/`）
5. 笔记名称 = 该组负向按同帖合并后的 `作者《标题》[*N]` 换行列表
