# Business Type Variants

ToB / ToC / ToD 三类业务在用户画像维度、满意度问卷维度、问题归属类别上有显著差异。本文件定义具体差异。

## 用户画像维度

### ToB(企业产品)

核心维度:
- **角色**:管理员、普通用户、决策者、执行者、审计/合规
- **职级**:基层执行、中层管理、高层决策
- **团队规模**:小型(<10)、中型(10-100)、大型(100+)
- **行业**:金融、零售、制造、医疗、政府、教育等
- **使用频率**:每日多次、每日、每周、按需

可视化建议:
- 角色 → 饼图
- 职级 → 条形图
- 团队规模 → 条形图
- 行业 → 条形图(如果行业种类多)或词云
- 使用频率 → 条形图

### ToC(消费者产品)

核心维度:
- **年龄段**:18-24、25-34、35-44、45-54、55+
- **地域**:一线/新一线/二线/三线及以下
- **使用场景**:工作、学习、娱乐、社交、购物
- **活跃度**:重度(每日多次)、中度(每日)、轻度(每周)、流失风险
- **设备偏好**:iOS / Android / Web
- **付费状态**:付费用户 / 免费用户 / 试用中

可视化建议:
- 年龄段 → 条形图
- 地域 → 饼图或地图(地图复杂度高,默认饼图)
- 使用场景 → 条形图
- 活跃度 → 条形图
- 设备偏好 → 饼图
- 付费状态 → 饼图

### ToD(开发者产品)

核心维度:
- **技术栈**:前端、后端、全栈、运维、数据、AI/ML、移动
- **编程经验**:< 1 年、1-3 年、3-5 年、5-10 年、10+ 年
- **职责**:开发、架构、技术管理、技术布道
- **团队规模**:个人开发者、小团队(<10)、中团队(10-50)、大团队(50+)
- **使用场景**:日常开发、生产部署、调试排错、性能优化
- **付费状态**:个人免费 / 团队 / 企业

可视化建议:
- 技术栈 → 饼图
- 编程经验 → 条形图
- 职责 → 饼图
- 团队规模 → 条形图
- 使用场景 → 条形图

9. 信息架构
10. 反馈机制

### ToC 维度集

1. 首次体验质量

7. 文案理解
8. 出错恢复
9. 信息架构
10. 反馈机制

### ToD 维度集

## 章节侧重差异

### ToB 报告应加重

- **角色协同效率章节**:跨角色交接、审批摩擦、信息透明度
- **配置正确率章节**:高风险操作的防错与校验有效性
- **可治理性章节**:权限、日志、审计、可追溯能力

### ToC 报告应加重

- **首次体验章节**:新用户首次完成关键路径的阻力点
- **转化漏斗章节**:关键节点流失原因与修复优先级
- **留存触发点**:哪些体验瞬间驱动复访

### ToD 报告应加重

- **学习成本章节**:新手到独立完成核心任务的时间
- **自助率章节**:用户在无人工协助时的问题解决比例
- **文档协同章节**:文档/示例/界面提示的闭环性

## 高风险任务识别标准

不同业务的"高风险"判断标准也有差异:

| 业务类型 | 高风险信号 |
|---|---|
| ToB | 完成率 < 70% 或 出错次数 > 1.5 或 涉及"误操作"风险 |
| ToC | 完成率 < 80% 或 满意度 < 6.0 或 在转化漏斗关键节点 |
| ToD | 完成率 < 70% 或 任务时长超出预期 50% 或 涉及"环境配置" |

## 业务类型与封面图建议

不同业务的封面图位用户上传内容也有不同建议:

- **ToB**:产品工作台截图、用户在办公场景使用的照片
- **ToC**:产品 App 截图、用户在生活场景使用的照片
- **ToD**:产品 IDE/Console 截图、代码截图、开发者工位照片

(此为给用户的提示,模型在询问"是否上传封面图"时可附带建议。)
