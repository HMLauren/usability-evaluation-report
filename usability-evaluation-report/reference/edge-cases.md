# Edge Cases · 边界与降级处理

材料缺失、样本异常、字段缺失、反推权限、隐私/图片质量等所有边界与降级规则的单一来源。**遇到材料缺失 / 异常 / 需要反推时,先读本文件再处理。**


### 用户没有 5 类材料中的某类

- 询问用户:这类材料确实没有(测试时没收集),还是暂时没整理好?
- 如果是前者,评估报告完整度——例如没有访谈材料,报告可以出但"用户原声"区块全部缺失,问题深挖只能基于观察记录定性描述
- 如果是后者,建议用户补齐再启动

### 中期快照(被试未到齐)

**检测与提示**:默认按正常报告跑,启动时不问终版/中期。材料盘点后若检测到**被试数明显少于计划**(例:画像表/任务书写了 8 人,聚类文件里只有 3 名被试),**提示一次**(话术见 `reference/dialog-scripts.md`「中期快照检测提示」)。N、M 由用户或材料给出,**不主动追问具体数值**,**只提示这一次**,用户不理会就按正常报告继续。

**中期口径**(用户确认是中期后才执行):

- 封面副标题、整体结论横幅、附录数据口径说明**三处**显式标注统一话术:
  > ⚠ 中期发现:本报告基于 N/M 名被试的初步数据,严重等级与统计指标待全样本完成后复核。
- 严重等级、统计指标的措辞从"是"改为"初步显示" / "本期数据显示"
- 文件名**不**加后缀(用户可能多次出中期报告,后缀增加管理负担;由用户自行命名版本)

### 被试数偏少 / 样本不充分

当模型判断被试数偏少时(由模型结合业务类型、研究主题、尼尔森公式综合判断,**不固定数值阈值**):

- 在屏 1.3 招募样本与样本充分性的核心观点中加 **"设计发现不充分,统计指标仅供参考"** 提示
- 不阻断报告生成
- 不要假装"样本充分"——研究员对样本判断的诚实是报告可信度的基础

### 观点高度集中在某一被试

当某一被试在观点聚类文件中的观点数显著高于其他被试时(由模型判断,**不固定百分比阈值**):

- 在附录"被试材料覆盖度登记表"下方加 **"样本偏差提示"**:某被试观点占比偏高,后续报告解读时建议留意该被试的代表性
- 不阻断报告生成
- 不擅自删减该被试的观点——保留数据完整,提示读者自行判断

### 观察记录缺失时的定量指标处理

当用户未提供观察记录时,任务完成率、出错次数、任务时长等指标:

- **仍然在 HTML 中显示该指标位**(保留版式完整,避免后期补数据需要重排)
- 数值位置显示 **"未采集"** 标记(灰色文字)
- 在该屏底部加一行小字注释:"* 任务完成率 / 出错次数 / 任务时长 等指标依赖现场观察记录,本期未采集"
- 目的:逼用户在下次研究中补齐,而不是默默隐藏

替代方案(可选):如果用户希望用"问题命中率"(影响被试数 / 总被试数)作为替代指标,模型可在用户明确要求时计算并标注 "* 数据来源:聚类观点统计,非现场观察"。

### JSON 字段缺失(步骤 2 已有规则)

- 个别观点缺 `原声摘取` → 标注 "(无原声引用)",不阻断
- 顶层只有 1 个元素(被试映射缺失) → 允许通过,在附录"数据口径说明"标注"被试映射缺失,被试 ID 沿用 JSON 原值"
- 结构性错误(顶层不是 array 等) → 拒绝,要求用户检查 usability-testing-interview-analysis 产物

### 反推规则(材料缺失时反推权限的单一来源)

材料缺失时,模型在不同维度上的反推权限:

| 缺失内容 | 允许反推 | 不允许反推 | 兜底处理 |
|---|---|---|---|
| 用户信息表 | 1) 先从问卷结果中找人口学/角色字段拼齐 2) 还不够则从观点聚类文件的 `用户` 字段反推被试姓名 + 数量 | 从产品名/逐字稿措辞反推职业/教育背景 | 画像页标注"画像信息待补充" |
| 任务书 | 从观点聚类文件的 `任务名称` 字段反推任务清单 | 凭空想象任务步骤、操作描述、来源依据 | 任务概览页标"任务详情待补充" |
| 问卷结果 | 从观点聚类文件中观点关联的任务数推算"问题命中率"(经用户确认) | 凭主观推算满意度分数 | 屏 2.1/2.2 相关指标位显示"待补充" |
| 风险等级 | 观点聚类文件已给 `严重程度` 时**直接用** | 观点聚类文件没给时**不要**擅自定级 | 标"待研究员复核" |
| 任务完成率等观察指标 | 用户明确要求用"问题命中率"替代时,模型可计算并加注释 | 不擅自用其他数据"凑"出完成率 | 见上"观察记录缺失"规则,显示"未采集" |

### 用户提供的图片质量不佳

不擅自处理。在中间态卡片中标注"图片质量待确认",让用户决定是否替换。

### 用户提供的逐字稿涉及隐私

询问用户:这份逐字稿是否已脱敏?如果未脱敏,提示用户应在报告中:
- 用 `P01 / P02 / ...` 替代真实姓名
- 涉及具体公司、产品、人名的部分用 `[公司]` `[产品]` `[姓名]` 替换
