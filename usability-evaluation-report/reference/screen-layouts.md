# Screen Layouts

每个独立的"事"(屏)按内容性质用对应版式。每屏 `min-height: 100vh`,内容紧凑布局,**不溢出**。

## 通用屏容器

```html
<section id="screen-id" class="screen">
  <div class="screen-header">
    <span class="screen-num">1.1</span>
    <h2 class="screen-title">屏标题(20-25 字内)</h2>
  </div>
  <p class="screen-lead">补充说明,1-2 句话(可选)</p>
  <!-- 内容区 -->
</section>
```

```css
.screen {
  min-height: 100vh;
  padding: 64px 48px;
  display: flex; flex-direction: column;
  border-bottom: 0.5px solid var(--line);
  max-width: 1280px;
  margin: 0 auto;
}
```

## 版式 1:封面(左右分栏)

**适用**:报告封面。

```
┌─────────────────────────┬──────────────────┐
│ 标签                    │                  │
│ 报告大标题              │   封面图占位     │
│ 副标题                  │   (用户上传)     │
│                         │                  │
│ ─────────────────       │                  │
│ 元信息(2 列网格)       │                  │
│ 产品 / 责任人           │                  │
│ 部门 / 更新时间         │                  │
└─────────────────────────┴──────────────────┘
```

栅格:1.1fr / 1fr,文字侧略宽。

## 版式 2:章节扉页

**适用**:每章开头,简介本章内容。

```
              ┌─────────────────┐
              │                 │
              │ 01 / 章节名称   │
              │                 │
              │ 章节大标题      │
              │                 │
              │ 章节简介文字    │
              │                 │
              └─────────────────┘
```

垂直居中,左对齐文字。`min-height: 50vh`(章节扉页可不满屏)。

```html
<section class="chapter-cover">
  <div class="num-big">01 / 研究设计</div>
  <h2>为什么测、测什么、怎么测</h2>
  <p>本章包含项目背景、评测场景与任务、招募样本与样本充分性。</p>
</section>
```

## 版式 3:左右分栏 + 照片网格

**适用**:研究背景、采样信息等需要现场照片支撑的内容。

```
┌─────────────┬────────────────┐
│ 文字 40%    │ 照片网格 60%   │
│             │ ┌────────────┐ │
│ 研究目的    │ │  主图(2x1)│ │
│ ...         │ ├──────┬─────┤ │
│ 研究方法    │ │ 图 2 │ 图 3│ │
│ ...         │ └──────┴─────┘ │
│ 采样信息    │                │
│ ...         │                │
└─────────────┴────────────────┘
```

栅格:1fr / 1.2fr。照片网格内部:2x2,主图占满第一行(`grid-column: span 2`)。

```html
<div class="bg-grid">
  <div class="bg-text">
    <h3>研究目的</h3><p>...</p>
    <h3>研究方法</h3><p>...</p>
    <h3>采样信息</h3><p>...</p>
  </div>
  <div class="bg-photos">
    <div class="bg-photo span-2">现场照片占位 · 主图</div>
    <div class="bg-photo">现场照片 2</div>
    <div class="bg-photo">现场照片 3</div>
  </div>
</div>
```

照片占位规则(背景页现场照片同样走图槽 + 文末图片清单,详见本文件「图片渲染与降级」):
- 这些位置各对应一个图槽(如 `img-bg1`、`img-bg2`…)。
- 图片清单状态 `已就绪` → 替换为 `<img src="data:image/...;base64,..." alt="[建议内容]" />`(单文件原则,base64 内嵌)。
- 状态 `待补充` → 保留虚线框 + 文字 = 该图槽的"建议内容" + " · 待补充"。

## 版式 4:数字侧栏 + 卡片网格(任务、问题等)

**适用**:评测任务清单、需求清单等"主体是多张卡片"的内容。

```
┌────────────┬──────────────────────┐
│ 数字侧栏   │ 卡片网格(2 列)     │
│ ┌────────┐ │ ┌──────┬──────┐     │
│ │  6     │ │ │ T01  │ T02  │     │
│ │ 任务   │ │ ├──────┼──────┤     │
│ ├────────┤ │ │ T03  │ T04  │     │
│ │  3     │ │ ├──────┼──────┤     │
│ │ 来源   │ │ │ T05  │ T06  │     │
│ ├────────┤ │ └──────┴──────┘     │
│ │  24    │ │                      │
│ │ 记录   │ │                      │
│ └────────┘ │                      │
└────────────┴──────────────────────┘
```

栅格:240px / 1fr。

每张任务卡的结构:
- 顶部:ID + 标题
- 底部:关键数据(完成率 / 出错次数 / 满意度)
- 悬停:浮起 + 边框变色
- 点击:跳转到对应问题详情(如果该任务有对应问题)

## 版式 5:数字侧栏 + 多图表网格(用户画像)

**适用**:用户画像页,展示岗位、经验、场景多个维度。

```
┌────────────┬──────────────────────┐
│ 数字侧栏   │ 图表网格(2x2)       │
│ ┌────────┐ │ ┌────────┬─────────┐ │
│ │  12    │ │ │ 岗位   │ 经验    │ │
│ │ 总数   │ │ │ (饼图)│ (条形) │ │
│ ├────────┤ │ ├────────┼─────────┤ │
│ │  7     │ │ │ 频率   │ 场景    │ │
│ │ 熟练   │ │ │ (条形)│ (条形) │ │
│ ├────────┤ │ └────────┴─────────┘ │
│ │  5     │ │                      │
│ │ 新手   │ │                      │
│ ├────────┤ │                      │
│ │ 98%    │ │                      │
│ │ 发现率 │ │                      │
│ └────────┘ │                      │
└────────────┴──────────────────────┘
```

栅格:280px / 1fr。

**图表数量自适应**:根据用户提供的画像维度数量,网格自动调整:

- 2 个图表 → 单行 2 列
- 3-4 个图表 → 2x2 网格
- 5-6 个图表 → 2x3 或 3x2 网格

```css
.persona-charts {
  display: grid;
  grid-template-columns: repeat(2, 1fr);  /* 默认 2 列 */
  gap: 12px;
}
/* 5+ 图表时改用 3 列 */
.persona-charts.many { grid-template-columns: repeat(3, 1fr); }
```

**适用**:每个独立的问题或需求,一个独立屏。

└──────────────────────────────────────────┘
┌────────────────────┬─────────────────────┐

## 屏间过渡

每屏之间用 `border-bottom: 0.5px solid var(--line)` 做轻分隔。
不要用大面积色块或全屏切换效果(避免太花哨)。

## 响应式策略

报告默认在 1280px 容器内展示,适配 1080p+ 显示器。

如果在窄屏(< 1024px)查看:

- 左右分栏改为上下堆叠
- 多图表网格降级为单列
- 问题切换 tab 横向滚动

但**不必在 mockup 阶段做响应式**,可以在 SKILL 中标注"如用户要求移动端友好,再补响应式 CSS"。

## 版式选择规则(内容性质 → 版式)

| 内容性质 | 推荐版式 |
|---|---|
| 封面 | 左右分栏(文字 + 大图) |
| 章节扉页 | 大编号 + 大标题 + 简介,垂直居中 |
| 项目背景 | 左右分栏(文字 40% + 照片网格 60%) |
| 任务概览 | 数字侧栏 + 任务卡网格 |
| 用户画像 | 数字侧栏 + 多图表网格(响应式,2-5 个图表自适应) |
| 结果总览 | 结论横幅置顶 + 雷达图 + 接纳率(双指标) |
| 单问题深挖 | 元信息条 + 左右双栏(现象/根因 \| 原声) + 优化建议横排 + 验收标准 |
| 单需求深挖 | 同问题深挖,但元信息字段不同 |
| 附录表格 | 筛选栏 + 全宽表格 |

## 一屏一事的紧凑度要求

- 一屏的内容必须在 1080p(1920×1080)显示器上**不滚动**就能看完。
- 内容多时**用网格布局**(两列、三列),不要单列垂直堆。
- 文字克制(标题 24px、正文 14-15px,不要堆 18px+ 大字号)。
- 留白适度,不松散也不挤压。
- 每个内容屏 `min-height: 100vh`,内容紧凑布局,**不溢出**。

## 图片渲染与降级

报告里的真实截图/照片,渲染时**只认卡片文末的「图片清单」表**(图槽ID → 建议内容 → 对应文件 → 状态),**不依赖"渲染器能不能看见图"**——这样文本小模型也能正确出图。表的样板与各列规则见 `card-templates.md`「图槽标记 & 文末图片清单表」。

**渲染输入的事实来源**:渲染以**卡片 md 为唯一事实来源**。用户可能离线编辑过卡片 md(改内容、调顺序,或在图片清单表里填了文件名)再回传——这种情况**重新完整读取回传的 md**、以它为准,不要用对话里的旧卡片版本。

**逐图槽渲染**:

- 状态 `已就绪`(对应文件到位)→ 把图放进该图槽位置;**默认 base64 内嵌**进单文件 HTML(单文件分发原则);`<img alt>` 用清单里的"建议内容"。
- 状态 `待补充` / 无对应文件 → 渲染明显的占位框(`border: 1px dashed; background: var(--bg-subtle, var(--bg-alt));`),框内文字 = 该图槽的"建议内容" + " · 待补充"。占位框**保留正确尺寸比例**,避免后期补图重排。
- **缺图绝不编造、不拿无关图顶替**,与数据"待补充"同等待遇。

**对账(渲染前轻量过一遍,都不阻断)**:

- 正文有图槽 ID、清单里没这行 → 按待补充处理,渲占位。
- 清单有行、正文找不到对应图槽 → 这张图无处可放,忽略(可在收尾提一句)。
- 清单"对应文件"写了名字、但用户没提供该文件 → 渲占位 + 标待补充。
- 以上都不报错、不阻断,与材料缺失同等对待。

**图多 / 单图过大的逃生通道**(不设死阈值,模型按"内嵌后体积是否影响打开/分发"综合判断,与 skill 整体"不设固定阈值"风格一致):

- 单张图过大 → 先**降采样 / 压缩**再 base64,控制单文件体积。
- 图很多、内嵌后单文件明显过大影响使用 → 改成 **HTML + 同级 `images/` 文件夹**(放弃单文件,改相对路径 `<img src="images/img-p03.png">`),并打包一并交付。此时若用户走离线编辑回传流程,`images/` 文件夹需随 md/html 一起带。
